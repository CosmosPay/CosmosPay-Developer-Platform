## Methods

### getAvailableWallets()

Probe and list wallets, flagging which are currently available.

```ts
getAvailableWallets(): Promise<WalletInfo[]>
```

**Returns** `Promise<WalletInfo[]>`

### registerWallet()

Register a custom wallet adapter (WalletConnect, Ledger, ‚Ä¶).

```ts
registerWallet(adapter: WalletAdapter, prepend?: boolean): this
```

**Parameters**

- `adapter` ‚Äî `WalletAdapter`
- `prepend` _(optional)_ ‚Äî `boolean`

**Returns** `this`

### disconnect()

Forget the current connection (and disconnect the adapter if it supports it).

```ts

```

**Returns** `Promise<void>`

### pay()

The one-call happy path: detect a wallet, adapt the intent into a transaction, sign it, and (unless told otherwise) broadcast it. Works for both `tx` intents (signs the returned XDR) and `pay` intents (builds the payment from the connected account).

```ts
pay(input: PayableInput, options?: PayOptions): Promise<PayResult>
```

**Parameters**

- `input` ‚Äî `PayableInput`
- `options` _(optional)_ ‚Äî `PayOptions`

**Returns** `Promise<PayResult>`

### buildTransaction()

- `input` ‚Äî `PayableInput`

### createPay()

Create a `pay` intent directly from the browser. Requires a browser-safe `apiKey` (trusted environments only). Accepts typed assets (`Assets.USDC`) and named addresses.

```ts

```

**Parameters**

- `options` ‚Äî `CreatePayPaymentIntentOptions`

**Returns** `Promise<PaymentIntentData>`
