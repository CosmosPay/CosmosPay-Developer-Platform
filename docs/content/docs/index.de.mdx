## Der Zahlungsablauf

Eine Zahlung besteht aus drei Schritten:

1. Der **Server** erstellt eine Absicht mit Ihrem geheimen API-Schlüssel (`client.paymentIntents.createPay`/`createTx`).
2. Der **Browser** schließt sie mit dem Stellar-Wallet des Nutzers ab (`new WebClient().pay(intent)`) und erzeugt einen On-Chain-`txHash`.
3. Der **Server** validiert diesen Hash (`intent.validate(txHash)`), wodurch der Status aktualisiert und ein Webhook ausgelöst wird.

Das Netzwerk wird durch den Typ des API-Schlüssels bestimmt: `dv_`-Schlüssel verwenden das **Testnet**, `prod_`-Schlüssel das **Mainnet**.
