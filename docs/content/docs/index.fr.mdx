## Le flux de paiement

Un paiement comporte trois étapes :

1. Le **serveur** crée une intention avec votre clé API secrète (`client.paymentIntents.createPay`/`createTx`).
2. Le **navigateur** la complète avec le portefeuille Stellar de l'utilisateur (`new WebClient().pay(intent)`), produisant un `txHash` sur la chaîne.
3. Le **serveur** valide ce hash (`intent.validate(txHash)`), ce qui met à jour le statut et déclenche un webhook.

Le réseau est choisi par le type de clé API : les clés `dv_` ciblent le **testnet**, les clés `prod_` ciblent le **mainnet**.
