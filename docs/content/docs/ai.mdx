These docs are built to be consumed by LLMs and coding agents (Cursor, Claude, ChatGPT…), not
just by humans. Copying the Markdown of a page is one option — but there's a lot more.

## Ready-made context files

Two endpoints expose the whole documentation as plain text an LLM can ingest in one shot:

- `/docs/llms.txt` — a compact index: every page with its title, URL and description.
- `/docs/llms-full.txt` — the full text of every page, concatenated.

```bash
# the index
curl https://dev.cosmospay.lat/docs/llms.txt

# everything, in one file (great for a single prompt or a RAG ingest)
curl https://dev.cosmospay.lat/docs/llms-full.txt
```

## Any single page as Markdown

Every page has an actions menu (top-right of the page) with:

- **Copy Markdown** / **View as Markdown** — the raw Markdown source of that page.
- **Open in ChatGPT** / **Open in Claude** — opens the assistant pre-loaded with that page.

You can also fetch a page's Markdown directly by appending `/content.md` under `/docs/llms.mdx`:

```bash
# the quickstart page, as Markdown
curl https://dev.cosmospay.lat/docs/llms.mdx/sdk/server/quickstart/content.md
```

<Callout title="Use it in a prompt">
Paste `llms-full.txt` (or a single page's Markdown) into your assistant and let it write the
integration — e.g. *"Using this, create a Cosmos Pay checkout that charges 10 USDC and verifies
the payment with a webhook."*
</Callout>

## The SDK ships its own LLM docs

The `@cosmosapp/pay_sdk` package bundles an AI-readable `llms/` folder, so once you install it
the docs are right there in `node_modules` — no network needed:

```text
node_modules/@cosmosapp/pay_sdk/llms/
├─ llms.txt          # index
├─ llms-full.txt     # everything in one file
└─ NN-*.md           # one atomic topic per file
```

```bash
# point your agent at the installed SDK docs
cat node_modules/@cosmosapp/pay_sdk/llms/llms-full.txt
```
