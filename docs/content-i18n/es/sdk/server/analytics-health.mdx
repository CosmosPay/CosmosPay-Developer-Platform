Estos endpoints de solo lectura te dan visibilidad operativa: cuánto has recaudado, la actividad reciente y si el servicio está en funcionamiento.

## Analíticas (solo lectura) — `client.analytics`

```ts
const summary  = await client.analytics.summary();   // totals, volume, webhook health, 30-day series
const balances = await client.analytics.balances();  // settled + pending per asset
const apiLogs  = await client.analytics.apiLogs();    // recent API requests
const hookLogs = await client.analytics.webhookLogs(); // recent webhook deliveries
```

`summary()` → `{ totals: { all, succeeded, pending, submitted, failed, cancelled,
expired, successRate }, volume: [...], webhooks: { endpoints, deliveries, ... } }`.

`balances()` → `{ data: [{ asset, amount, pending, count }], total }`
donde `amount`/`pending` son cadenas decimales y `count` es la cantidad de pagos.

## Salud (sondas públicas) — `client.health`

```ts
await client.health.liveness();        // → true when the service is up
const ready = await client.health.readiness(); // → { status, info, details } (checks the DB)
```

Estas rutas no requieren un consumidor autenticado.
