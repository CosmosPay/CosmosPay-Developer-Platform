## El flujo de pago

Un pago tiene tres pasos:

1. El **servidor** crea una intención con tu clave de API secreta (`client.paymentIntents.createPay`/`createTx`).
2. El **navegador** la completa con la billetera de Stellar del usuario (`new WebClient().pay(intent)`), produciendo un `txHash` en cadena.
3. El **servidor** valida ese hash (`intent.validate(txHash)`), lo que actualiza el estado y dispara un webhook.

La red se elige según el tipo de clave de API: las claves `dv_` apuntan a **testnet**, las claves `prod_` apuntan a **mainnet**.
