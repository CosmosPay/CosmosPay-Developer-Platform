---
title: "Autenticação"
description: "Autentique-se com uma única chave de API; a rede (testnet vs mainnet) é escolhida automaticamente pelo prefixo da chave."
---

A autenticação é deliberadamente mínima: uma única chave de API secreta, mantida no seu servidor. O prefixo da chave também seleciona a rede Stellar por você, então não há mais nada a configurar. É novo por aqui? Veja [como obter sua chave de API](/api-keys).

**Você só precisa da sua chave de API.** Todo o resto (URL do gateway, versão da API) faz
parte da configuração compartilhada do SDK, mantida pelo Cosmos Pay.

```ts
import { Client } from '@cosmosapp/pay_sdk';

const client = new Client({ apiKey: process.env.COSMOS_PAY_API_KEY });
```

A chave de API é enviada como `Authorization: Bearer <apiKey>`.

## Seleção de rede (importante)

A **rede é determinada pelo tipo da chave** — você NÃO a configura:

| Prefixo da chave | Rede |
| ---------- | ------- |
| `dv_…`     | testnet |
| `prod_…`   | mainnet |

Por causa disso, os emissores de ativos são específicos de cada rede: use `Assets.*` para a mainnet
e `TestnetAssets.*` para a testnet (veja a documentação de ativos).

## Mantenha a chave no servidor

A chave de API é um segredo. Crie e valide intenções no seu **servidor**. O
cliente web no navegador apenas assina/envia; ele não precisa da chave. (Ele *pode* receber
uma `apiKey` para protótipos confiáveis, mas nunca envie uma chave de produção para o navegador.)

## Sobrescritas compartilhadas / de auto-hospedagem (avançado)

```ts
// Change the gateway for every client created afterward (once, at startup):
Client.shared.baseURL = 'https://gateway.your-domain.com';

// Or per-client (rarely needed) — e.g. local dev bypassing the gateway:
const client = new Client({
  apiKey: '...',
  baseURL: 'http://localhost:3000',
  gatewaySecret: process.env.APISIX_GATEWAY_SECRET,
  consumerUsername: 'cosmos_demo',
});
```

`baseURL`, `version`, `gatewaySecret`, `consumerUsername` são todos opcionais — uma
integração normal nunca os define.
