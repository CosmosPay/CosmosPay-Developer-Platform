## O fluxo de pagamento

Um pagamento tem três etapas:

1. O **servidor** cria uma intenção com sua chave de API secreta (`client.paymentIntents.createPay`/`createTx`).
2. O **navegador** a conclui com a carteira Stellar do usuário (`new WebClient().pay(intent)`), produzindo um `txHash` on-chain.
3. O **servidor** valida esse hash (`intent.validate(txHash)`), o que atualiza o status e dispara um webhook.

A rede é escolhida pelo tipo de chave de API: chaves `dv_` usam a **testnet**, chaves `prod_` usam a **mainnet**.
